# xrun v0.2 — TUI (ratatui frontend, live updates, command palette)

## Overview

Поднять `xrun-tui` крейт поверх готового `xrun-core`/`xrun-vast`/`xrun-poller`.
По умолчанию `xrun` без аргументов открывает TUI; работают экраны Runs / Run detail
(Stages, Logs, Manifest tabs) / Launch picker / Instances / Settings.
Биндинги по `docs/TUI.md`. Live update — через `mpsc::channel` от поллера к
TUI-event loop. SQLite остаётся source of truth, TUI пишет в БД только через
`xrun-core` API. Acceptance: можно запустить новый run, посмотреть как обновляются
стадии, спулить best ckpt не выходя из TUI.

## Context

- **Files involved**: новый крейт `crates/xrun-tui/`, изменения в `xrun-cli` для
  `xrun` без args → TUI.
- **Reference docs**: `docs/TUI.md` (экраны, биндинги, виджеты), `docs/STATE.md`
  (что читать), `docs/EVENTS.md` (формат событий для Stages-tab).
- **Внешние зависимости**: `ratatui = "0.29"`, `crossterm = "0.28"`,
  `tokio = "1"` (rt-multi-thread, sync, time, signal), `tui-input = "0.10"`,
  `throbber-widgets-tui = "0.7"`, `tui-logger = "0.13"`, `color-eyre = "0.6"`.
  Опционально: `tachyonfx = "0.10"` (только если есть простор; в этом плане НЕ используем).
- **Не используем**: `tui-realm` (overhead), сторонние chart-крейты (Metrics tab — в v0.3).
- **Архитектура event loop**: tokio `select!` между `crossterm::event::EventStream`,
  каналом обновлений данных (`mpsc<DataUpdate>`) и периодическим `tokio::time::interval`
  для перерисовки на 60Hz max. Render → `terminal.draw(|f| view::render(f, &state))`.
- **State**: `AppState` — иммутабельная при чтении, мутируется только в одном таске.
  Включает `Vec<RunSummary>`, `Option<RunDetail>` (если открыт detail), фокус,
  модалки, тему. Source of truth — БД; State кэширует и инвалидируется на
  `DataUpdate::RunsChanged` / `DataUpdate::RunUpdated(run_id)`.
- **Тема**: дефолт low-contrast greys + accent. High-contrast — переключение через
  Settings, сохраняется в `GlobalConfig.tui.theme` (новое поле).

## Development Approach

- **Testing approach**: для view-функций — snapshot тесты через
  `ratatui::backend::TestBackend` (рендер в фиксированный buffer, сравнение).
  Для логики state-machines — unit-тесты на `apply(event)`.
- Без `unsafe`. Без panics в render (any error → render error pane).
- **CRITICAL: every task MUST include new/updated tests.**
- **CRITICAL: all tests must pass before starting next task.**

## Out of scope

## Anti-patterns

- Не использовать `tokio::task::block_in_place` в render — блокирует event loop. Все IO — async через каналы.
- Не плодить state-машины в каждом screen — единая `AppState` с дискретным enum.
- Не вызывать `vendor.execute` напрямую из TUI — только через `xrun-core` API.
- Не писать в SQLite из TUI напрямую — только через `xrun-core::store`.
- Не делать polling БД быстрее 1Hz — все обновления реактивны через канал.
