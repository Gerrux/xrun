# xrun v0.2.1 — Vendors screen, auth flow, balance wiring, splash

## Overview

Закрыть три долга, накопившиеся в v0.2:
1. **Аутентификация вендоров** — сейчас единственный способ ввести ключ это
   `xrun config set vast.api_key …` или ручная правка
   `~/.config/xrun/credentials.toml`. После того как `xrun` без аргументов
   открывает TUI, первое впечатление — пустой экран без подсказки что делать.
2. **Реальный баланс** — `view/status_bar.rs` показывает заглушку `$—`,
   мокап `docs/TUI.md:10` обещает `balance: $34.12`.
3. **Видимость вендоров** — TUI знает только `vendor` как строку в строке Run
   и не показывает: настроен ли вендор, его баланс, последние ошибки
   соединения, доступные офферы.

Дополнительно — лёгкий splash-оверлей при старте, чтобы first-run не выглядел
как пустая таблица.

## Development Approach

- Backend изменения (xrun-core, xrun-vast) — с unit-тестами на парсинг
  ответа `show user`, на `import_from_native`, на масочный рендер.
- TUI слой — без новых внешних крейтов; используем `tui-input` пока **не**
  тащим (overhead для одного поля); вместо этого простой буфер строки
  + render `••••` для секретов.
- Splash — реализуется как `Modal::Splash` с deadline-полем; авто-исчезает
  через таймер в основном цикле `app::event_loop`.
- VendorRegistry в xrun-tui — внутренняя простая мапа `HashMap<String,
  Box<dyn VendorAdapter>>`. Регистрация в `App::new` по доступным
  credentials. Для не-настроенного вендора — нет адаптера, статус
  «not configured».
- **Validation gate**: `cargo check --workspace` после каждой crate-task,
  `cargo clippy --workspace -- -D warnings` + `cargo test --workspace`
  в финале.

- **Offers viewer** — отдельной таблицей доступных машин. Решено: войдёт

## Anti-patterns

- Не показывать ключи в plaintext в TUI — только маска `••••`.
- Не вызывать `cli::*` функции из xrun-vast напрямую из xrun-tui минуя
  `VendorAdapter` trait. Регистрация адаптеров в одном месте (`App::new`).
- Не блокировать `event_loop` на probe — всё через фоновый поток + канал.
- Не складировать введённый ключ в state дольше чем модалка открыта; на close
  модалки — обнулять `EditField.value`.
- Не показывать splash повторно на каждый запуск, только при first-run.
