# xrun v0.3 — MLflow client, Kaggle adapter, Metrics tab with charts

## Overview

Закрыть MLflow-зеркалирование (числовые ряды + figure-артефакты) и второй вендор
Kaggle. После этого:
- метрики из тренировки автоматически появляются в MLflow UI; ссылку можно
  открыть из TUI/CLI;
- `xrun metrics --png` рисует график (через MLflow figure API + локальный
  fallback на `plotters`);
- Metrics tab в TUI показывает live-чарт через `ratatui::widgets::Chart` с
  multi-series toggle;
- `xrun launch exp/foo.yaml` с `vendor: kaggle` отрабатывает end-to-end: push
  kernel → polling status → ingest output → метрики и события восстановлены в
  ту же БД, отображаются рядом с vast-runs.

## Context

- **Files involved**: новые крейты `crates/xrun-mlflow/`, `crates/xrun-kaggle/`,
  изменения в `xrun-poller`, `xrun-tui`, `xrun-cli`, `python/xrun_hook/`.
- **Reference docs**: `docs/EVENTS.md` (Kaggle fallback), `docs/MANIFEST.md`
  (kaggle секция), `docs/CLI.md` (`xrun metrics --png`, `--mlflow-url`),
  `docs/TUI.md` (Metrics tab спека).
- **Внешние зависимости (Rust)**:
  `reqwest = { version = "0.12", default-features = false, features = ["json", "rustls-tls"] }`,
  `plotters = "0.3"` (только backend `bitmap` + features `png`),
  `url = "2"`, `base64 = "0.22"` (для basic auth), `tempfile = "3"` (тесты).
- **MLflow версия**: REST API 2.x (совместимо с `mlflow >= 2.0`). Endpoints:
  `/api/2.0/mlflow/experiments/get-by-name|create`, `/runs/create|update|log-batch`,
  `/runs/log-metric`, `/runs/log-param`, `/artifacts/list`. Auth: optional
  `notify_token` через `Authorization: Bearer <token>` (если `mlflow.token` в
  credentials), либо basic auth (`mlflow.username/password`).
- **MLflow availability**: если `mlflow.url` не задан в `GlobalConfig` ИЛИ
  `manifest.mlflow` отсутствует ИЛИ REST вернул network error → silent
  degrade на «SQLite-only» (с warning в `tracing` 1 раз на run, не на каждый
  log-batch).
- **Kaggle CLI**: используем `kaggle` Python CLI через subprocess (как vastai).
  `kaggle kernels push -p <dir>` (требует `kernel-metadata.json`),
  `kaggle kernels status <slug>`, `kaggle kernels output <slug> -p <dir>`.
- **Kaggle xrun_hook delivery**: `enable_internet=false` запрещает pip install.
  Решение: при `vendor: kaggle` адаптер кладёт wheel `xrun_hook-X.Y.Z-py3-none-any.whl`
  в kernel directory и в head-cell ноутбука/скрипта инжектит:
  `import sys, glob; sys.path.insert(0, glob.glob('xrun_hook-*.whl')[0])`.
  Для скриптов (`run.cmd: python script.py`) wrapper-скрипт `_xrun_kaggle_entry.py`
  делает sys.path-инъекцию + import + execfile target.
- **Kaggle artefact path**: на Kaggle всё что в `/kaggle/working/` — преобразуется
  в kernel output. Hook пишет в `/kaggle/working/run/`. После `complete` адаптер
  скачивает весь output, парсит `events.jsonl`/`metrics.jsonl` post-hoc.

## Development Approach

- **Testing approach**: TDD для MLflow REST клиента (используя `wiremock`),
  для парсинга Kaggle JSON-выводов и для PNG-генерации (snapshot-test
  по байтам PNG-файла после reduce — сравнение размеров и crc32).
- E2E с реальным Kaggle/MLflow — только под env-gate `XRUN_KAGGLE_SMOKE=1` /
  `XRUN_MLFLOW_SMOKE=1`, по дефолту `#[ignore]`.
- **CRITICAL: every task MUST include new/updated tests.**
- **CRITICAL: all tests must pass before starting next task.**

## Out of scope (для v0.4+)

- `xrun sweep` — декартово произведение grid.
- `xrun diff <a> <b>` — сравнение manifest+метрик.
- Anomaly detection (loss spike → notification).
- Native vast REST вместо CLI subprocess.
- Web UI рядом с TUI.
- Kaggle live-tail через REST (workaround `[xrun-event]` маркер в stdout — best effort).

## Anti-patterns

- Не блокировать поллер на ошибках MLflow — degrade silently после первого warning.
- Не парсить kaggle stdout как plain text где есть `-m` (machine-readable).
- Не аплоадить чекпоинты в MLflow — только маленькие artefacts (PNG, JSON metric summary).
- Не писать MLflow client в каждом крейте — единый `xrun-mlflow`.
- Не зашивать MLflow URL в код — всегда из `GlobalConfig.mlflow.url`.
- Не делать live-tail для Kaggle — это ложное ожидание; явно документировать «no live data until completion».
