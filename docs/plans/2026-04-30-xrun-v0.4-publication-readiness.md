## Stage 2 — CLI: закрыть exit-64 stubs (2-3 h)

### 2.1 `xrun logs --follow`

**Файл:** `crates/xrun-cli/src/commands/logs.rs`

Текущее поведение: возвращает exit 64 «not supported».  
Нужно: tail stdout.log файла из `runs/<id>/stdout.log` (на локальном диске).
Для живых runs — файл обновляется poll-daemon'ом.

Реализация:
- Если run активен → открыть файл, seek to end, poll каждые 500ms (`tokio::time::interval`)
- Если run завершён → просто вывести содержимое и выйти
- `--since 10m` — offset по timestamp в строках (формат `[HH:MM:SS]`)
- `--grep <pat>` — regex фильтр на строке

### 2.2 `xrun events --follow`

**Файл:** `crates/xrun-cli/src/commands/events.rs`

Нужно: SELECT из SQLite + poll новых строк по run_id каждые 1s.  
Завершить когда `runs.status IN (done, failed, cancelled)`.

## Stage 4 — CLAUDE.md в корне (1 h) ← КРИТИЧНО

**Файл:** `CLAUDE.md` (создать)

CLAUDE.md — это инструкция для Claude Code о проекте. Должна содержать:

```markdown
# xrun — ML experiment runner

## Что это
Rust CLI + Python Textual TUI для запуска ML-экспериментов на vast.ai и Kaggle.
Один YAML-манифест → provision → upload → run → poll events/metrics → SQLite.

## Стек
- `xrun` (Rust binary): CLI, launch, ls, show, logs, events, metrics, pull, stop, rerun, balance, config, doctor
- `python/xrun_tui/`: Python Textual TUI (запускается через `xrun tui` или `xrun` без аргументов)
- `xrun_hook` (Python package): пишется в training script для эмита events.jsonl + metrics.jsonl
- SQLite: `~/.local/share/xrun/runs.db` (Linux/Mac) / `%APPDATA%/xrun/runs.db` (Windows)

## Сборка и запуск
cargo build --release          # билд бинаря
target/release/xrun doctor     # проверить окружение
target/release/xrun config init

# TUI (Python)
cd python/xrun_tui && pip install -e .
xrun tui   # или просто xrun без аргументов

## Ключевые файлы
- docs/CLI.md — полный справочник CLI
- docs/MANIFEST.md — YAML-схема эксперимента
- docs/TUI.md — экраны и биндинги TUI
- docs/ARCHITECTURE.md — компоненты и поток данных
- docs/SKILL.md — дизайн Claude-скилла

## Запуск тестов
cargo test --workspace
cargo clippy --workspace -- -D warnings

## Манифесты экспериментов
Лежат в exp/*.yaml. Запуск: xrun launch exp/foo.yaml [--detach]

## Что НЕ делать
- Не вызывать vastai/kaggle CLI напрямую — только через xrun
- Не читать runs.db через sqlite3 — только через xrun ... --json
- Не писать в events.jsonl / metrics.jsonl вручную
```

**Done when:** `cat CLAUDE.md` даёт Claude Code достаточно контекста без чтения

Взять за основу `docs/SKILL.md`, но это должен быть **runtime файл** (краткий,
практический), а не дизайн-документ. Содержание:

```markdown
# xrun skill

xrun — ML experiment runner. Используй его для всех запусков на vast.ai/Kaggle.

## Быстрые команды

xrun launch exp/foo.yaml --detach    # запустить эксперимент
xrun ls [--status running] [--json]  # список запусков
xrun show <id>                       # карточка запуска
xrun logs <id> [--follow]            # stdout
xrun events <id> [--follow]          # стадии (provision/upload/train/done)
xrun metrics <id> [--key val_f1] [--ascii] [--json]
xrun pull <id> [--ckpt best] [--into models/]
xrun stop <id>
xrun rerun <id> [--patch run.args.--lr=5e-4]
xrun balance                         # баланс vast.ai
xrun doctor                          # проверка окружения

## Типичный flow

1. Создать/скопировать манифест в exp/
2. xrun launch exp/foo.yaml --detach
3. xrun ls --status running --json   → взять run_id
4. xrun events <id> --follow         → следить за стадиями
5. xrun metrics <id> --ascii         → смотреть метрики
6. xrun pull <id> --ckpt best        → забрать чекпоинт

## Anti-patterns

НЕ делай:  vastai create instance / kaggle kernels push / sqlite3 runs.db
Делай:     xrun launch / xrun ls --json / xrun metrics --json

## Манифест (минимальный)

name: my_experiment
vendor: vast
gpu: RTX_4090
data:
  - src: data/train.h5
    dst: /workspace/data/train.h5
run:
  cmd: python train.py
  args:
    --lr: 5e-4
    --batch-size: 4
artifacts:
  patterns: ["checkpoints/best*.pt"]

## --json везде

Все read-команды поддерживают --json. Парси через python -c "import json,sys; ..."
или jq для принятия решений.
```
